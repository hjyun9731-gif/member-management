## 관리번호 규칙

| 구분 | 형식 | 시작 | 예시 |
|------|------|------|------|
| 신규등록 | 신YY-N | 1 | 신26-1, 신26-2 |
| 양도양수 회원 | 양YY-N | 1 | 양26-1, 양26-2 |
| 폐업 | 폐-N | 80 | 폐-80, 폐-81 |
| 양도(폐지) | 양-N | 28 | 양-28, 양-29 |
| 이관 | 이-N | 4 | 이-4, 이-5 |

- 연도(YY)가 바뀌면 신규/양도양수 번호는 1부터 다시 시작
- 폐지 번호는 연도 없음 (폐-80 형식 유지)
- 자동 생성되지만 수정 가능

---

## 개인/택배 자동 구분

차량번호에 **"배"** 가 포함되면 택배, 없으면 개인으로 자동 판단
- 예) `배12가1234` → 택배회원
- 예) `강원12가1234` → 개인회원
- 사용자가 직접 선택하지 않아도 됨

## 주요 업무 플로우

### 예정자 → 신규등록
1. 회원관리 > 예정자/양도양수 클릭
2. "예정자" 탭에서 정보 입력 후 **저장**
3. 예정자 목록에서 **✅** 클릭
4. 인가일자 입력 + 관리번호 확인(수정 가능)
5. **등록 완료** → 개인/택배 회원으로 자동 등록

### 양도양수 → 회원 등록
1. 회원관리 > 예정자/양도양수의 **"양도양수"** 탭
2. 정보 입력(인가일자, 양도정보 포함) 후 **저장**
3. 양도양수대장에 자동 기록 + 회원 자동 등록

### 회원 폐업 처리
1. 개인회원 또는 택배회원 목록에서 **폐업** 버튼 클릭
2. 폐업 / 양도 / 이관 중 선택
3. 처리일자 입력 + 관리번호 확인
4. 폐업 처리 완료 → 폐지현황에 자동 저장
5. 해당 회원은 목록에서 제거

## 엑셀 업로드

### 지원 파일 종류
- 면허자현황 (xlsm/xlsx)
- 양도양수대장
- 폐지현황
- 변경이력대장
- 부과대수

### 업로드 방법
1. 엑셀 업로드 > 파일 업로드 클릭
2. **파일 종류 선택** (필수)
3. 중복 처리 방식 선택 (건너뛰기/덮어쓰기/새로 추가)
4. 파일 드래그 또는 클릭해서 선택
5. **미리보기** → 데이터 확인
6. **업로드 확정** → DB 저장

### 컬럼 자동 매칭
프로그램이 엑셀 컬럼명을 자동으로 인식합니다.
매칭 안 되는 컬럼은 raw_data에 보존됩니다.

1. GitHub에 코드 push

## 오류 해결

| 오류 | 해결 |
|------|------|
| `venv\Scripts\activate` 실패 | PowerShell에서 `Set-ExecutionPolicy RemoteSigned` 실행 |
| `pip install` 실패 | `py -m pip install -r requirements.txt` 시도 |
| 포트 충돌 | `.env` 파일에서 `PORT=8001` 변경 |
| DB 오류 | `member_management.db` 삭제 후 재시작 |
| 데이터 사라짐 | `member_management.db` 파일 백업 필수 |
